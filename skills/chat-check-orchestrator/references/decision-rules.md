# 决策规则

仅定义编排门控，不展开 API 或执行实现细节。

1. 信息缺失时，先补问一个最关键槽位，不做推荐。
2. 信息足够但高置信直执行条件未达（无论候选数量）时，只能进入推荐/确认等门控流程，不可擅自执行。若信号不足，应先让用户确认当前候选是否就是目标对象，或在多候选中明确选择后再推进。
3. 仅当满足高置信直执行条件时，允许直执行。
   高置信判定锚定候选返回信号：`rank_score`、`sample_count`，以及前两名 `rank_score` 的差值（派生判断）。
4. 候选为空时必须停下确认，不允许臆测执行目标。
5. 候选分数接近时必须停下确认，避免误命中。
